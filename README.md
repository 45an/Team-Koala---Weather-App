A simple Weather App that provides weather information for Swedens capital Stockholm. The application consists of a RESTful API built with ASP.NET Core, Xunit Testing and a React client application. The API is thoroughly tested with unit tests to ensure the accuracy and reliability of the weather data.

## Features

- Get weather information for Swedens capital Stockholm
- Display temperature, humidity, wind speed, and city name
- Responsive and visually appealing UI

- Node.js (v12 or higher)

### Backend Setup

1. Clone the repository.
2. Open the `WeatherApp.sln` solution file in Visual Studio or your preferred IDE.
3. Build and run the application.


### Frontend Setup

1. Open a terminal and navigate to the `WeatherClient` directory.
2. Install the dependencies by running the command: `npm install`.
3. Start the React development server: `npm run dev`.

The application should now be up and running. Open your browser and navigate to `http://localhost:20030` to see the Weather App.

## üñ•Ô∏è API Documentation

The API provides the following endpoints:

### GET /weather/{city}

Returns the current weather information for the specified city.

Example Response:

```json
{
  "city": "Stockholm",
  "temperature": 22,
  "humidity": 70,
  "wind": 10,
}

```


GET /time

Returns the current time.

Example Response:

```json
{
  "hour": 15,
  "minute": 30,
  "second": 45
}

```

GET /health

Returns a simple health check response to verify if the API is running.

Example Response: 
```json

API is running.

```

## üìà Xunit Testing

The Weather App API is extensively tested with unit tests to ensure the accuracy and reliability of the weather data. 
The test file WeatherForecastTest.cs contains the following tests:

WeatherEndpoint_ReturnsWeatherData: Verifies that the `/weather/{city}` endpoint returns the correct weather data for a specific city.
TimeEndpoint_ReturnsCurrentTime: Verifies that the `/time endpoint` returns the current time.
HealthEndpoint_ReturnsOk: Verifies that the `/health` endpoint returns a successful status.



## üíª React Application

The React application, WeatherClient, consumes the Weather App API to display weather information for Swedens capital
Stockholm.
It uses Axios for making HTTP requests and displays the weather details on a card component.
Client Application Structure

App.jsx: This file contains the main React component that fetches weather data from the Weather App API and renders it on the card component.

App.css and index.css: These files contain CSS styles for the WeatherClient application.

public/index.html: This file serves as the HTML template for the React application.

### Running the React Application

To run the React client application, follow these steps:

    Make sure the Weather App API is running.
    Open a terminal in the WeatherClient application directory.
    Run the command npm install to install the dependencies.
    Run the command npm start to start the development server.
    The application will be accessible at http://localhost:20300.
